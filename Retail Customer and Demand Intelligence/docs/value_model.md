# Value Model — Annual Profit Impact (illustrative)

Turns the four analysis levers into money. **Revenue bases are computed from the real
artifacts** (segments, member status, top-100 SKUs) and annualised by the ~2.02-year data
span; **uplift assumptions are explicit and sensitivity-tested.** All figures are in the
proxy's GBP and are **illustrative decision-framing numbers, not the retailer's actuals** — the
data has no COGS, and lost sales are not directly observable.

## Assumptions (base case)
| Assumption | Value |
|---|---|
| Gross margin | 35% (sensitivity 25% / 35% / 45%) |
| Protect — retained revenue lift on high-value base | +2pp |
| Loyalty — Non-Member revenue converted / spend uplift | 10% / +5% |
| Rescue — incremental return rate vs control | +12pp (~experiment MDE) |
| Rescue — one-time coupon depth / comeback order | 30% / £396 |
| Grow — cross-sell response / ARPU uplift on responders | 10% / +15% |
| Availability — top-100 revenue recovered via fewer stockouts | +2% |

## Annual profit impact (base case, margin 35%)
| Lever | Annual profit (GBP) |
|---|---|
| Protect — retention of high-value base | £41,400 |
| Availability — forecast + newsvendor stocking | £20,000 |
| Rescue — win-back, net of coupon | £11,400 |
| Grow — cross-sell ARPU uplift | £6,800 |
| Protect — loyalty / Non-Member conversion | £2,200 |
| **Total** | **≈ £81,800 / yr** |

Total ≈ **0.8% of annual product revenue (~£9.7M)** — modest in absolute terms because
the proxy is a ~£10M/yr business and the assumptions are deliberately conservative; **the
same percentages scale linearly** at the retailer's revenue.

## Ranges
- **Scenario** (scale all uplifts ×0.5 / ×1 / ×1.5): £40.9k — £81.8k — £122.7k / yr.
- **Margin sensitivity** (25% / 35% / 45%): £57.4k — £81.8k — £106.2k / yr.

## What the model says
**Retention (Protect) is the single biggest lever**, ahead of availability and the
targeted CRM plays — consistent with the thesis that defending the high-value base beats
chasing marginal sales. The value model's job is to **rank the levers and frame the
investment transparently**, not to claim a precise number.

## Honest caveats
- No COGS → margin is an assumption (sensitivity-tested).
- Lost sales are unobservable → the Availability lever is the most assumption-heavy.
- Uplifts are planning assumptions; the **experiments (holdout controls) exist to replace
  them with measured effects** before scaling spend.


## Rescue campaign economics (illustrative, treated group)

Reported on the **treated group**, not diluted across all customers — the way a campaign is actually judged:

| | |
|---|---|
| Targeted (high-value, lapsing) | 250 customers |
| Expected incremental win-back (+12pp vs control) | **~30 customers** |
| Revenue recovered (~£1,427/customer/yr) | **~£43k** |
| Margin recovered (35%) | **~£15k** |
| Coupon cost (30% on a ~£396 comeback order) | ~£3.6k |
| **Net profit** | **~£11k** |
| **Return on campaign spend** | **~3×** |

Expected figures; the randomised control group measures the *true* lift before scaling.
